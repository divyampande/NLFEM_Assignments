# Finite Deformation Kinematics Solver

This directory contains a custom Python-based Finite Element solver designed to compute the large-deformation kinematics of a rigidly constrained composite structure subjected to non-uniform heating. 

The solver uses a 4-node linear isoparametric element formulation to map a reference domain to a highly deformed physical domain, calculating the Deformation Gradient and resulting strain fields.

## File Overview

* **`verification.py`**: A data pre-processing script. It takes raw, noisy node coordinates extracted via WebPlotDigitizer and enforces strict mathematical boundaries, horizontal symmetry, and centerline constraints to produce a clean dataset.
* **`non_uniform_heating.py`**: The main finite element solver. It reads the cleaned coordinates, constructs the mathematical Jacobian for the shape functions, computes the kinematic tensors, and generates the final contour plots.
* **`corrected_deformed_coords.csv`**: The cleaned coordinate data generated by the verification script.
* **`results/`**: An auto-generated directory where all output plots (`.png`) and computed tensor arrays (`.npy`) are saved.

## How to Run

1. **Clean the Data**: If you are providing new raw coordinates, run `verification.py` first to generate the `corrected_deformed_coords.csv` file.
2. **Execute the Solver**: Run `non_uniform_heating.py`. 

### Configuration Options
Inside `non_uniform_heating.py`, you can modify the `COMPUTE_MODE` variable at the top of the script to change how the strains are calculated and visualized:
* `COMPUTE_MODE = "element"`: Computes constant strain values at the geometric center of each element and plots them as distinct rigid blocks.
* `COMPUTE_MODE = "nodal"`: Computes strains at the Gauss points, performs nodal averaging for strain recovery, and plots a smoothed continuous contour field.

## Outputs
The script will automatically create a `results/` folder and populate it with:
* High-resolution visualizations of the deformed mesh and the Green-Lagrange strain fields (E11, E22, E12).
* Raw computed tensor data saved as `.npy` files for the Deformation Gradient (F), Green-Lagrange Strain (E), Eulerian Strain (e), and Engineering Strain.
